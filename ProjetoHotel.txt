Exercício: Sistema de Reservas de Hotéis - API Complexa

Nível: Avançado

O objetivo deste exercício é criar uma API RESTful para um sistema de reservas de hotéis. A API permitirá que os usuários vejam os hotéis disponíveis, façam reservas, gerenciem reservas existentes e muito mais.

Modelagem de Dados:

Crie classes de modelo para representar os principais componentes do sistema:

Hotel: Representa um hotel com informações como nome, endereço, número de quartos, etc.
Room: Representa um quarto dentro de um hotel.
Reservation: Representa uma reserva feita por um usuário para um quarto específico em um determinado período.

Hotel:
	- Id
	- Nome
	- Endereço
	- Quartos
		- Id
		- NumeroQuarto
		- Tipo
			- Quarto Standart
			- Quarto Deluxe
			- Suite Presidencial
		- Dispoibilidade
	- Reserva
		- Id
		- QuartoId
		- Entrada
		- Saida
		- StatusReserva


Endpoints da API:

Crie endpoints RESTful para as seguintes operações:

1- Cadastrar um hotel
2- Listar todos os hotéis disponíveis.
3- Visualizar detalhes de um hotel específico.
4- Listar os quartos disponíveis em um hotel.
5- Fazer uma reserva para um quarto em um hotel.
6- Visualizar detalhes de uma reserva específica.
7- Listar todas as reservas de um usuário.
8- Cancelar uma reserva.


Autenticação e Autorização:

Implemente autenticação de usuários com JWT (JSON Web Token).
Defina papéis (roles) como "user" e "admin" para diferentes níveis de acesso.

Validações e Tratamento de Erros:

Realize validações de entrada adequadas para garantir que os dados inseridos sejam válidos. Lidere com erros de maneira apropriada, retornando respostas com códigos e mensagens claras.

Paginação e Filtragem:

Implemente paginacão e filtros para a listagem de hotéis, quartos e reservas.

Documentação da API:

Utilize uma ferramenta como o Swagger para documentar a API, incluindo detalhes sobre os endpoints, parâmetros, autenticação, exemplos de solicitação e resposta.

Testes Automatizados:

Crie testes unitários e de integração para garantir que a API funcione corretamente.

Gestão de Estado:

Implemente mecanismos para gerenciar o estado de quartos (disponíveis, ocupados, em manutenção) e o status de reservas (confirmadas, canceladas etc.).

Notificações:

Implemente notificações por e-mail para confirmações de reservas, atualizações de status e outros eventos relevantes.
